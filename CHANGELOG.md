# 開発ログ — JPHACKS ハッカソン版からの改善記録

> このドキュメントは、JPHACKS 2025 ハッカソンで作成した姿勢検出アプリを、個人開発で**ゼロからWebアプリとして再構築**し、段階的に改善した記録です。

---

## ハッカソン版の状態（Before）

| 項目 | 内容 |
|---|---|
| 動作環境 | macOS のみ（`afplay`, `osascript` 等のOS固有コマンドに依存） |
| 実行方法 | ターミナルで Python スクリプトを実行（`python detector.py`） |
| UI | OpenCV のウィンドウに映像を表示するだけ |
| 検出指標 | 肩の傾き1種類（固定しきい値 15° で判定） |
| 通知 | OS通知 + システムビープ音 |
| 対応デバイス | PC（外付けカメラ前提） |
| コード量 | 約130行（Python 1ファイル） |

**課題:**
- Windows/スマホでは動作しない
- しきい値が固定のため、体格やカメラ位置によって精度が変わる
- ブラウザで使えないため、配布・共有が困難

---

## 現在の状態（After）

| 項目 | 内容 |
|---|---|
| 動作環境 | **ブラウザ（PC・スマホ）** — OS不問 |
| 実行方法 | HTMLファイルを開くだけ / GitHub Pages でURL共有可能 |
| UI | ダークテーマ、ステータスバー、レスポンシブ対応 |
| 検出指標 | **4種類**（肩の傾き・頭の傾き・前のめり・猫背） |
| 基準設定 | **キャリブレーション方式**（ユーザーごとに基準を自動測定） |
| 通知 | Web Audio API による警告音 + 画面赤フラッシュ（姿勢が直るまで繰り返し） |
| 対応デバイス | PC・スマホ（レスポンシブ対応済み） |
| バックエンド | FastAPI による REST API（記録の保存・取得） |
| コード量 | 約800行（HTML + CSS + JS + Python） |

## 改善の詳細

### 1. ブラウザベースのWebアプリとして再構築

**なぜ:** Python + OpenCV はインストールが必要で、非エンジニアには配布できない。ブラウザなら URL を共有するだけで誰でも使える。

**やったこと:**
- HTML/CSS/JavaScript でフロントエンドを一から構築
- MediaPipe Pose の JavaScript 版（CDN経由）に移行し、サーバー不要で姿勢検出を実現
- `getUserMedia` API でブラウザからカメラに直接アクセス

**技術的ポイント:**
- MediaPipe の Camera ユーティリティは解像度を 640×480 に固定してしまうため、`getUserMedia` + `requestAnimationFrame` による独自の検出ループを実装。デバイス本来のカメラ比率を維持できるようにした

---

### 2. キャリブレーション方式の導入

**なぜ:** ハッカソン版は「肩の傾き 15° 以上で警告」という固定しきい値だった。しかし体格・座り方・カメラの高さは人それぞれ違うため、ある人には敏感すぎ、別の人には鈍すぎる問題があった。

**やったこと:**
- スタート時に 3秒間ユーザーの良い姿勢を測定し、その平均値を「基準（baseline）」として保存
- 以降はこの基準からのズレで判定するため、体格やカメラ位置に自動で適応する
- いつでも基準を取り直せる「再調整ボタン」も実装

---

### 3. 検出指標を1種類 → 4種類に拡張

**なぜ:** 肩の傾きだけでは「前のめり」「猫背」「頭だけ傾いている」といった姿勢の崩れを検出できない。

**追加した指標:**

| 指標 | 検出する姿勢の崩れ | 計算方法 |
|---|---|---|
| 肩の傾き (`shoulderTilt`) | 肩の左右の傾き | 左肩・右肩を結ぶ線の角度 |
| 頭の傾き (`headTilt`) | 頭・首の傾き | 左耳・右耳を結ぶ線の角度 |
| 前のめり (`shoulderWidth`) | 画面への近づきすぎ | 肩幅の変化（近づくと大きくなる） |
| 猫背 (`neckRatio`) | 背中が丸まっている | (肩の高さ - 耳の高さ) / 肩幅 の比率 |

---

### 4. 時間ベースのアラート制御

**なぜ:** ハッカソン版は姿勢が崩れた瞬間に即通知だった。しかし物を取るなど一瞬の動きでも鳴ってしまい、実用上ストレスになる。

**やったこと:**
- 悪い姿勢が **3秒以上持続** して初めて警告を発動（一瞬の動きは無視）
- 姿勢が悪い間は **4秒ごとに繰り返し** 警告音 + 画面フラッシュ
- 姿勢を直した瞬間にタイマーをリセットし、警告を即停止
- Web Audio API で音を生成するため、外部の音声ファイルが不要

---

### 5. 鏡表示の実装

**なぜ:** カメラ映像をそのまま表示すると左右が反転し、「右肩が下がっています」と言われても直感的にどちらか分からない。

**やったこと:**
- Canvas の `translate + scale(-1, 1)` で映像を左右反転（自撮りと同じ見え方）
- 判定メッセージの左右テキストもこの反転に合わせて調整

---

### 6. レスポンシブ対応（スマホ対応）

**なぜ:** ノートPCのカメラでは顔しか映らないことがあり、スマホのカメラの方が上半身を映しやすい場面がある。

**やったこと:**
- `@media (max-width: 480px)` でスマホ用レイアウトを実装
- ボタンを縦並びにし、タッチ操作しやすいサイズに拡大
- Canvas の高さを `auto` にし、デバイスのカメラ比率をそのまま反映

---

### 7. FastAPI によるバックエンド API

**なぜ:** 姿勢の記録を蓄積し、改善の推移を可視化するため。また、今後 Gemini API 等の外部 API を安全に呼ぶにはバックエンドが必要（API キーをフロントに置くとユーザーに漏れるため）。

**やったこと:**
- FastAPI + Pydantic で REST API を構築（`POST /records`, `GET /records`）
- CORS ミドルウェアでフロントエンドからのアクセスを許可
- Swagger UI (`/docs`) で API の動作確認が可能

---

### 9. セッション記録機能

**なぜ:** 使いっぱなしでは改善の実感がない。セッションごとに「良い姿勢/悪い姿勢の時間」を記録し、スコアとして可視化することで改善のモチベーションにつなげたい。

**やったこと:**
- フロントエンドで毎フレームの経過時間を良い/悪いに振り分けて計測
- ストップ時にスコア（良い姿勢の割合 %）・時間・コメントをパネルで表示
- FastAPI に自動送信し、SQLite で永続保存（サーバー未起動でもフロント単体で動作）
- バックエンドをインメモリ → SQLite に移行（サーバー再起動してもデータが消えない）
